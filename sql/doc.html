Overview

The SQL Query Generator is a lightweight web tool that creates SQL scripts (INSERT, UPDATE, DELETE) from tabular data. You can paste rows directly into the tool or upload a CSV / Excel file (.csv, .xls, .xlsx). The first row is expected to contain column names, and subsequent rows are treated as data rows.

Features

  • Generate INSERT, UPDATE, and DELETE statements.
  • Paste tabular text (tab or comma separated) or upload CSV / Excel.
  • Configure WHERE column(s) for UPDATE / DELETE using & (AND) and || (OR).
  • Unicode support for string literals (prefix strings with N'...').
  • Copy or download generated SQL file.
  • Editable table cells after paste or file render — you can correct values inline.

How to use

1. Configure table & query type

Enter the target Table Name and choose query type: INSERT, UPDATE, or DELETE.

2. Choose input method

Use Paste Data (default) — click into the paste area and paste tabular text (first row = column names). Or choose Upload CSV/Excel and select a file (.csv, .xls, .xlsx). The tool uses the first worksheet when reading Excel.

3. (For UPDATE / DELETE) Specify WHERE column(s)

If you're creating UPDATE or DELETE statements, enter the WHERE columns. See the WHERE logic section for syntax.

4. Use Unicode (optional)

Toggle the Use Unicode checkbox if your text values must be prefixed with N'...' (SQL Server Unicode string literal).

5. Generate and export

Click Generate SQL. Your SQL will appear in the output box. Use Copy SQL to copy to clipboard or Download SQL to save a .sql file. You can edit the pasted table before generating — the generator always reads the current table content.

WHERE clause syntax (UPDATE / DELETE)

Specify one or more columns that the generator will use to compose the WHERE clause for each data row. The special syntax lets you build AND / OR combinations:

  • Single column: CODE → WHERE CODE = value
  • AND: join columns with & (ampersand) → CODE&SR_NO → WHERE CODE = v1 AND SR_NO = v2
  • OR groups: separate groups with || (double pipe) → CODE&SR_NO||CODE → WHERE (CODE = v1 AND SR_NO = v2) OR (CODE = v1)

Important: Column names must exactly match the first (header) row values. If a column in the WHERE expression is not found, the generator will alert you.

Unicode option

Toggle Use Unicode to wrap string values as N'Your string' (SQL Server Unicode literal). This is helpful for international characters.

Behavior notes:

  • Null-like values (empty or the literal NULL) are preserved as NULL without quotes.
  • Numeric values are not quoted.
  • Single quotes inside strings are escaped (doubled) before the value is quoted.

Examples

INSERT example

Header row: CODE,NAME,AMOUNT
Data row: ABC,Test,100

INSERT INTO Menus (CODE, NAME, AMOUNT) VALUES ('ABC', 'Test', 100);

UPDATE example

WHERE columns: CODE&SR_NO

UPDATE Menus
SET CODE = 'ABC', NAME = 'Changed', AMOUNT = 200
WHERE CODE = 'ABC' AND SR_NO = 1;

DELETE example

WHERE columns: CODE

DELETE FROM Menus WHERE CODE = 'ABC';

Complex WHERE (OR groups)

Input: CODE&SR_NO||CODE

WHERE (CODE = 'ABC' AND SR_NO = 1) OR (CODE = 'ABC')

Implementation details (how the tool works)

  1. Input parsing: Pasted text is split by newline into rows, then by tab or comma into columns.
  2. Header row: First row is treated as column names and used for SQL column lists and mapping.
  3. Value formatting:
    • Empty or literal NULL → NULL
    • Numeric values → left unquoted
    • Strings → escaped and quoted; Use Unicode toggles N'...'
  4. UPDATE: All columns are placed in the SET clause; WHERE clause built per-row from the specified expression.
  5. DELETE: WHERE clause built per-row from the specified expression.
  6. Output: All generated statements are concatenated into a single SQL text blob which the user can copy or download.

Limitations & notes

  • The tool expects a correct header row — column matching is exact and case-sensitive depending on your DB mapping.
  • Data type inference is basic: numbers vs strings. For special types (dates, GUIDs) you may need to post-process the generated SQL.
  • Large files may be slow in the browser for very large row counts (thousands+). For heavy loads, use server-side conversion.
  • The tool does not execute the SQL — it only generates script text. Always review and run in a safe test environment first.

FAQ & Troubleshooting

Q: My WHERE column is not found — what to do?

Ensure the column name exactly matches a header cell (no extra spaces). If your header contains whitespace, copy it exactly or edit the header cell in the paste table to a clean name.

Q: My numbers are being quoted as strings

If a numeric value contains spaces, thousand separators, or non-digit characters, it's treated as a string. Clean the input or edit the cell before generating.

Q: Excel upload shows strange separators

Excel is converted to CSV internally. If values contain commas, ensure they were stored properly in Excel — the converter will preserve cell boundaries when reading the sheet.

Q: Can I use functions or raw SQL in a cell?

Values are quoted/escaped automatically. If you want raw SQL expressions (e.g., GETDATE()) instead of quoted strings, you must edit the generated SQL manually after generation.

Troubleshooting checklist

  • Verify header row exists and matches WHERE columns.
  • Edit table cells directly (header or data) if parsing looks off.
  • Use the Clear button to reset and try again.
  • Test generated SQL in a safe non-production database first.

Shortcuts & Pro tips

  • Paste from Excel directly — Excel copies are tab-separated and will paste cleanly into the tool.
  • Edit header names in-place to match your database column names before generating.
  • Use OR groups (||) to fall back on alternate keys when composing WHERE clauses.
